# Appu Kaju — Premium Dry Fruit Experience

A production-grade, cinematic ecommerce experience for **Appu Kaju** (Lucknow, since 1998), in an earthy editorial style — parchment, forest green and terracotta with big Zodiak serif headlines. The signature moment: a photographed brand pouch that **tilts open on first scroll and pours real photographed kernels** through six ingredient chapters before the Premium Mix finale.

## Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 15 (App Router, RSC) · React 19 · TypeScript strict |
| Styling | Tailwind CSS v4 (design tokens in `app/globals.css` `@theme`) |
| Micro/UI animation | Framer Motion (`motion` v12) |
| Scroll choreography | GSAP 3 + ScrollTrigger, synced with Lenis smooth scroll |
| State | Zustand (+ `persist`) — cart, wishlist, recently viewed |
| Forms | React Hook Form + Zod |
| Fonts | Zodiak (serif display) + Satoshi (body) + Clash Display (numerals/eyebrows), self-hosted woff2 via `next/font/local` |

**Division of labor:** GSAP owns scroll-driven timelines · Framer Motion owns enter/hover/exit/layout. No WebGL — the hero is photographic, so it cannot jank or crash on weak GPUs.

### The hero, briefly
One pinned 600vh ScrollTrigger drives the story. The pouch is two photo layers (`public/images/cutouts/packet-lid.png` + `packet-base.png`, generated by `scripts/make-cutouts.mjs`: Pollinations source → sharp edge-flood background keying → crisp SVG label composited on). On first scroll the pouch tilts into a pour pose (easeOutBack overshoot) and the lid peels open (`rotateX`, local perspective). Each of the six chapters then pours a time-based GSAP burst of photographed kernel cutouts out of the mouth — gravity easing, drift, tumble, depth scale — which land in a persistent pile while an editorial panel presents the ingredient's benefits from `lib/data/ingredients.ts`. Continuous values are set imperatively via `gsap.set` on refs; React state changes only at chapter boundaries, so scrolling never re-renders per frame. Scroll-back clears later piles; fast-scrolling back-fills earlier piles instantly.

**Fallback:** `prefers-reduced-motion` renders `hero-fallback.tsx` — the same story as static editorial sections.

### Theme morphing
`ThemeZone` interpolates `--page-bg` / `--page-fg` CSS vars as chapters scroll into view — the page background travels chocolate → cream → forest → cream → chocolate without any section seams.

### Images
All imagery is generated free via Pollinations.ai: `scripts/fetch-images.mjs` (site photography, rerun-safe seed cache) and `scripts/make-cutouts.mjs` (hero pouch + kernel cutouts with sharp background keying). `ProductVisual` renders every image on a branded gradient stage derived from the product's accent color and degrades gracefully if a file is missing. Replace any file in `public/images/**` with real photography and it appears automatically.

## Ecommerce (demo mode)
Cart/wishlist persist in localStorage. Checkout validates with Zod across 3 animated steps and writes the order to `sessionStorage` for the confirmation page. **No payment is processed.** To go live: create a Razorpay order server-side in a route handler, mount Razorpay checkout in step 3 of `app/checkout/page.tsx`, and verify the signature in a confirmation handler.

## SEO / a11y / performance
- Metadata API on every route, `Organization`/`Product`/`FAQPage` JSON-LD, sitemap, robots, dynamic OG image.
- Radix primitives, focus-visible rings, skip link, `sr-only` hero narrative, reduced-motion respected globally (static `hero-fallback.tsx`).
