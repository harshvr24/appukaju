# Appu Kaju — Premium Dry Fruit Experience

A production-grade, cinematic ecommerce experience for **Appu Kaju** (Lucknow, since 1998), in an earthy editorial style — parchment, forest green and terracotta with big Zodiak serif headlines. The signature moment: a pinned **"table film" hero** — eight moody full-bleed photographs (an opening table spread, six ingredient close-ups, the Premium Mix) crossfade like a slow food film as you scroll, with serif captions, benefit icons and per-product CTAs.

## Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 15 (App Router, RSC) · React 19 · TypeScript strict |
| Styling | Tailwind CSS v4 (design tokens in `app/globals.css` `@theme`) |
| Micro/UI animation | Framer Motion (`motion` v12) |
| Scroll choreography | GSAP 3 + ScrollTrigger, synced with Lenis smooth scroll |
| State | Zustand (+ `persist`) — cart, wishlist, recently viewed |
| Forms | React Hook Form + Zod |
| Fonts | Zodiak (serif display) + Satoshi (body) + Clash Display (numerals/eyebrows), self-hosted woff2 via `next/font/local` |

**Division of labor:** GSAP owns scroll-driven timelines · Framer Motion owns enter/hover/exit/layout. No WebGL — the hero is photographic, so it cannot jank or crash on weak GPUs.

### The hero, briefly
One pinned 640vh ScrollTrigger plays eight stacked full-bleed photographs (`public/images/cinematic/*.webp`, generated by `scripts/fetch-cinematic.mjs`) as a scroll-scrubbed film: each scene fades in over the last with a slow Ken Burns drift, while editorial captions (serif title, Hindi name, benefit lines with themed icons from `components/home/benefit-icon.tsx`, an Explore CTA resolved from `lib/data/products.ts`) animate in per scene. Whole photographs only — no alpha-keyed cutouts, no 3D — so nothing can fringe or crash. Continuous values are set imperatively via `gsap.set` on refs; React state changes only at scene boundaries. The hero also flips `--header-fg` to parchment while its dark stage is pinned.

**Fallback:** `prefers-reduced-motion` renders `hero-fallback.tsx` — the same story as static editorial sections.

### Theme morphing
`ThemeZone` interpolates `--page-bg` / `--page-fg` CSS vars as chapters scroll into view — the page background travels chocolate → cream → forest → cream → chocolate without any section seams.

### Images
All imagery is generated free via Pollinations.ai: `scripts/fetch-images.mjs` (site photography) and `scripts/fetch-cinematic.mjs` (the hero's moody table-film set) — both rerun-safe via seed/marker caches. `ProductVisual` renders every image on a branded gradient stage derived from the product's accent color and degrades gracefully if a file is missing. Replace any file in `public/images/**` with real photography and it appears automatically.

## Ecommerce (demo mode)
Cart/wishlist persist in localStorage. Checkout validates with Zod across 3 animated steps and writes the order to `sessionStorage` for the confirmation page. **No payment is processed.** To go live: create a Razorpay order server-side in a route handler, mount Razorpay checkout in step 3 of `app/checkout/page.tsx`, and verify the signature in a confirmation handler.

## SEO / a11y / performance
- Metadata API on every route, `Organization`/`Product`/`FAQPage` JSON-LD, sitemap, robots, dynamic OG image.
- Radix primitives, focus-visible rings, skip link, `sr-only` hero narrative, reduced-motion respected globally (static `hero-fallback.tsx`).
